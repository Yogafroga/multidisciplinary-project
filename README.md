# multidisciplinary-project

# /backend
- # /app
  - ## /api
    Папка с эндпоинтами (роутерами). Здесь организуются маршруты (routes), которые принимают HTTP-запросы, обрабатывают их, вызывают бизнес-логику из сервисов и возвращают ответы клиенту.
    Каждый эндпоинт отвечает за конкретную функцию API, например, регистрацию пользователя, авторизацию, работу с сущностями и т.д.

  - ## /migration
    Папка, содержащая миграции базы данных. В ней хранятся скрипты обновления структуры БД, созданные с помощью Alembic. 
    Миграции обеспечивают версионное управление схемой БД, позволяя безопасно вносить изменения и откатывать их.

  - ## /models
    Папка с ORM-моделями. Здесь описываются классы моделей, соответствующие таблицам в базе данных. Модели определяют структуру данных и связи между таблицами. Именно эти классы используются SQLAlchemy для работы с записью и чтением данных из БД.

  - ## /services
    Папка для бизнес-логики приложения. В сервисах реализуются правила обработки данных, взаимодействие с моделями и репозиториями, выполнение операций, которые не связаны напрямую с http-запросами. Такой слой помогает разделять логику и контролировать сложность приложения.
  **Для каждого эндпоинта в /api лучше создавать соответствующий файл в /services (для лучшей навигации ну и чтоб понятно было что где лежит).**

- # schemas
    Папка с Pydantic-моделями – схемами валидации и сериализации данных. Они описывают форматы входящих запросов и выходящих ответов, обеспечивают проверку данных и служат документацией API.
